# spec.md — Priority Analysis

## 목표

> Mail Summary와 Mail Classification 결과를 기반으로 이메일의 중요도와 긴급도를 분석하여 사용자가 먼저 확인해야 하는 메일인지 판단한다.

---

## 맥락

### 대상

* 대학생
* 직장인
* 이메일을 자주 사용하는 일반 사용자

### 사용 목적

중요한 메일을 빠르게 식별하여
사용자가 우선적으로 확인해야 하는 메일을 추천한다.

---

## 범위

### 입력

* Mail Summary 결과
* Mail Classification 결과
* 메일 제목
* 메일 본문
* 발신자
* 수신 시간

### 출력

Markdown 형식의 중요도 분석 결과

포함 내용

* 메일 정보
* 중요도(1~5점)
* 긴급도
* 중요도 판단 이유
* 우선 확인 여부

---

## 제약

* 중요도는 반드시 1~5점으로 표시한다.
* 중요도 판단 이유를 반드시 작성한다.
* 메일을 삭제하거나 이동하지 않는다.
* 중요도는 메일 내용, 발신자, 마감일, 답장 필요 여부를 종합하여 판단한다.
* Markdown 형식을 유지한다.
- 모든 출력은 공통 출력 형식의 "다음 Skill 전달 정보" 섹션을 포함한다.
- 다음 Skill이 사용할 수 있도록 핵심값을 명확한 항목명으로 작성한다.
- 없는 정보는 추측하지 않고 "없음" 또는 "확인 필요"로 표시한다.
---

---

## 중요도 점수 기준

Priority Score는 1~5점으로 계산한다.

기본 점수는 1점에서 시작한다.

### 가산 기준

| 조건 | 점수 |
|---|---:|
| 오늘 마감 또는 24시간 이내 처리 필요 | +3 |
| 3일 이내 마감 | +2 |
| 7일 이내 마감 | +1 |
| 답장이 필요한 메일 | +2 |
| 학교/업무 관련 메일 | +2 |
| 교수, 팀원, 회사, 공식 기관 등 중요 발신자 | +2 |
| 금융, 결제, 계정, 보안 관련 메일 | +2 |
| 첨부파일 확인이 필요한 메일 | +1 |
| 일정 또는 회의가 포함된 메일 | +1 |

### 감점 기준

| 조건 | 점수 |
|---|---:|
| 광고/프로모션 메일 | -2 |
| 일반 뉴스레터 | -1 |
| 단순 알림 메일 | -1 |
| 이미 처리된 메일 | -2 |

### 점수 보정

- 최종 점수는 1점 미만으로 내려가지 않는다.
- 최종 점수는 5점을 초과하지 않는다.
- 여러 조건에 해당하면 점수를 합산한 뒤 1~5 범위로 보정한다.

### 중요도 등급

| 최종 점수 | 등급 | 의미 |
|---:|---|---|
| 5 | 매우 높음 | 즉시 확인 필요 |
| 4 | 높음 | 오늘 안에 확인 권장 |
| 3 | 보통 | 여유 있을 때 확인 |
| 2 | 낮음 | 나중에 확인 가능 |
| 1 | 매우 낮음 | 광고/일반 알림 가능성 높음 |

## 출력 형식

파일명

PriorityAnalysis.md

섹션 순서

1. 메일 정보
2. 중요도
3. 긴급도
4. 중요도 판단 이유
5. 우선 확인 여부

분량

200~400자
