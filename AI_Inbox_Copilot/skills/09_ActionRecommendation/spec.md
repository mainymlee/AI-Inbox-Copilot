# spec.md — Action Recommendation

## 목표

> 이메일 분석 결과를 종합하여 사용자가 지금 수행해야 하는 행동을 우선순위와 함께 추천한다.

---

## 맥락

### 대상

* 대학생
* 직장인
* 이메일을 자주 사용하는 일반 사용자

### 사용 목적

사용자가 이메일를 읽은 후
무엇을 해야 하는지 빠르게 판단하도록 돕는다.

---

## 범위

### 입력

* Mail Summary 결과
* Mail Classification 결과
* Priority Analysis 결과
* Security Check 결과
* Safe Archive 결과

### 출력

Markdown 형식의 행동 추천

포함 내용

* 추천 행동
* 행동 우선순위
* 행동 이유
* 사용자 확인 필요 여부
* 주의사항

---

## 제약

* 실제 메일을 삭제하지 않는다.
* 실제 답장을 전송하지 않는다.
* 실제 캘린더에 등록하지 않는다.
* 실제 메일을 이동하지 않는다.
* 모든 행동은 추천만 제공한다.
* Markdown 형식을 유지한다.
- 모든 출력은 공통 출력 형식의 "다음 Skill 전달 정보" 섹션을 포함한다.
- 다음 Skill이 사용할 수 있도록 핵심값을 명확한 항목명으로 작성한다.
- 없는 정보는 추측하지 않고 "없음" 또는 "확인 필요"로 표시한다.
---

## 출력 형식

파일명

ActionRecommendation.md

섹션 순서

1. 추천 행동
2. 행동 우선순위
3. 행동 이유
4. 사용자 확인 필요 여부
5. 주의사항

분량

250~500자

---

## 성공 기준

* [ ] 사용자에게 필요한 행동을 추천한다.
* [ ] 행동 우선순위를 제공한다.
* [ ] 행동 이유를 설명한다.
* [ ] 사용자 확인 필요 여부를 표시한다.
* [ ] 위험한 행동은 경고와 함께 제공한다.
* [ ] Morning Mail Brief에서 사용할 수 있는 결과를 생성한다.
