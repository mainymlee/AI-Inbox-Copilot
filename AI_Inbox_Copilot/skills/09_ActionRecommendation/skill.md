--------------------------------------------------------------------

## 목적 및 범위

Mail Summary, Mail Classification, Priority Analysis, Security Check, Safe Archive 결과를 종합하여 사용자가 수행해야 할 행동을 추천한다.

결과물

* ## 추천 행동
* ## 행동 우선순위
* ## 행동 이유
* ## 사용자 확인 필요 여부
* ## 주의사항

5개 섹션으로 구성된 `.md` 파일

---

## 작업 절차

1. spec.md에서 목표, 출력 형식, 성공 기준을 읽는다.
2. Mail Summary Skill 결과를 읽는다.
3. Mail Classification Skill 결과를 읽는다.
4. Priority Analysis Skill 결과를 읽는다.
5. Security Check Skill 결과를 읽는다.
6. Safe Archive Skill 결과를 읽는다.
7. 메일의 중요도, 보안 위험도, 보관 정책을 종합한다.
8. 사용자에게 필요한 행동을 우선순위에 따라 결정한다.
9. 행동 이유를 작성한다.
10. 사용자 확인이 필요한 행동인지 판단한다.
11. Markdown 형식으로 결과를 생성한다.
12. 성공 기준과 결과물을 비교하여 누락된 항목이 없는지 확인한다.
13. 다음 Skill이 사용할 수 있도록 "다음 Skill 전달 정보" 섹션을 작성한다.
14. 정보가 불확실한 경우 "확인 필요"로 표시한다.
15. 성공 기준과 결과물을 비교하여 누락된 항목이 없는지 확인한다.
---

## 사용 규칙 및 제약 사항

* 실제 메일을 삭제하지 않는다.
* 실제 답장을 전송하지 않는다.
* 실제 캘린더 등록을 수행하지 않는다.
* 실제 메일 이동을 수행하지 않는다.
* 모든 결과는 추천 형태로 제공한다.
* 보안 위험이 높은 메일은 경고 문구를 포함한다.
* Safe Archive 정책을 따른다.
* Markdown 형식을 유지한다.

---

## 템플릿

```markdown
# ✅ Action Recommendation

## 추천 행동

- 답장 작성
- 일정 등록
- 지금 확인
- 나중에 확인
- 광고함 보관
- 위험 메일함 검토
- 비밀번호 변경
- 공식 홈페이지 확인

---

## 행동 우선순위

**높음 / 중간 / 낮음**

---

## 행동 이유

- 메일 중요도가 높음
- 마감일이 임박함
- 피싱 위험이 존재함
- 답장이 필요함

---

## 사용자 확인 필요 여부

- 필요
- 불필요

---

## 주의사항

- 링크를 클릭하기 전에 발신자를 확인하세요.
- 첨부파일 실행 전 안전성을 확인하세요.
- 중요한 메일은 삭제하지 마세요.
```
