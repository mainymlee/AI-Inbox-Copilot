-------------------------------------------------------------------------

## 목적 및 범위

Mail Summary와 Mail Classification의 분석 결과를 기반으로
이메일의 중요도와 긴급도를 판단한다.

결과물

* ## 메일 정보
* ## 중요도
* ## 긴급도
* ## 중요도 판단 이유
* ## 우선 확인 여부

5개 섹션으로 구성된 `.md` 파일

---

## 작업 절차

1. spec.md에서 목표, 출력 형식, 성공 기준을 읽는다.
2. Mail Summary Skill의 결과를 읽는다.
3. Mail Classification Skill의 결과를 읽는다.
4. 메일 제목과 요약 내용을 분석한다.
5. 중요도 점수 기준에 따라 기본 점수 1점에서 시작하여 가산/감점 조건을 적용한다.
   - 오늘 마감 또는 24시간 이내 처리 필요: +3
   - 3일 이내 마감: +2
   - 7일 이내 마감: +1
   - 답장이 필요한 메일: +2
   - 학교/업무 관련 메일: +2
   - 중요 발신자: +2
   - 금융/결제/계정/보안 관련 메일: +2
   - 첨부파일 확인 필요: +1
   - 일정 또는 회의 포함: +1
   - 광고/프로모션 메일: -2
   - 일반 뉴스레터: -1
   - 단순 알림 메일: -1
   - 이미 처리된 메일: -2
6. 긴급도를 높음, 중간, 낮음으로 판단한다.
7. 중요도 판단 이유를 작성한다.
8. 사용자가 즉시 확인해야 하는지 여부를 결정한다.
9. 성공 기준과 결과물을 비교하여 누락된 항목이 없는지 확인한다.
10. 다음 Skill이 사용할 수 있도록 "다음 Skill 전달 정보" 섹션을 작성한다.
11. 정보가 불확실한 경우 "확인 필요"로 표시한다.
12. 성공 기준과 결과물을 비교하여 누락된 항목이 없는지 확인한다.
---

## 사용 규칙 및 제약 사항

* 중요도는 반드시 1~5점으로 표시한다.
* 중요도 판단 이유를 반드시 작성한다.
* 객관적인 정보만을 기반으로 판단한다.
* 메일을 삭제하거나 이동하지 않는다.
* Security Check Skill에서 활용할 수 있도록 명확한 결과를 제공한다.
* Markdown 형식을 유지한다.
- 중요도는 반드시 중요도 점수 기준을 적용하여 계산한다.
- 최종 점수는 1~5 범위로 보정한다.
- 점수 산정에 사용한 조건을 중요도 판단 이유에 명시한다.

## 템플릿

```markdown
# ⭐ Priority Analysis

## 메일 정보

- 발신자:
- 제목:
- 수신 시간:

---

## 중요도

- 기본 점수: 1
- 가산/감점 조건:
  - 오늘 마감: +3
  - 학교/업무 관련: +2
  - 광고/프로모션: 해당 없음
- 최종 점수: 5/5
- 중요도 등급: 매우 높음

---

## 다음 Skill 전달 정보

- priority_score: 5
- priority_level: VERY_HIGH
- urgency_level: HIGH
- reply_required: 확인 필요
- deadline: 확인 필요
- should_check_now: true

## 긴급도

- 높음 / 중간 / 낮음

---

## 중요도 판단 이유

- 마감일이 존재함
- 답장이 필요함
- 업무/학교 관련 메일
- 중요 발신자로부터 수신

---

## 우선 확인 여부

✅ 즉시 확인 필요

또는

⏳ 나중에 확인 가능
```
