# spec.md — Safe Archive

## 목표

> AI의 오탐 가능성을 고려하여 이메일을 즉시 삭제하지 않고, 메일 유형과 위험도에 따라 안전한 보관 정책을 제안한다.

---

## 맥락

### 대상

* 메일 자동 정리를 원하지만 중요한 메일 삭제를 걱정하는 사용자
* 광고, 뉴스레터, 피싱 의심 메일이 많이 쌓이는 사용자

### 사용 목적

메일을 자동으로 삭제하지 않고 일정 기간 보관하여
오탐으로 인한 중요한 메일 손실을 방지한다.

---

## 범위

### 입력

* Mail Classification 결과
* Priority Analysis 결과
* Security Check 결과
* Newsletter Manager 결과
* 메일 제목
* 발신자
* 수신일

### 출력

Markdown 형식의 안전 보관 정책

포함 내용

* 추천 보관함
* 보관 기간
* 삭제 가능 여부
* 사용자 확인 필요 여부
* 보관 이유

---

## 제약

* 어떤 경우에도 메일을 즉시 삭제하지 않는다.
* 삭제는 사용자 최종 확인 후 가능하다고 안내한다.
* 피싱 의심 메일은 위험 메일함에 격리 보관을 추천한다.
* 광고 메일은 광고함에 일정 기간 보관을 추천한다.
* 중요 메일은 받은편지함 유지를 추천한다.
* Markdown 형식을 유지한다.
- 모든 출력은 공통 출력 형식의 "다음 Skill 전달 정보" 섹션을 포함한다.
- 다음 Skill이 사용할 수 있도록 핵심값을 명확한 항목명으로 작성한다.
- 없는 정보는 추측하지 않고 "없음" 또는 "확인 필요"로 표시한다.
---

## 출력 형식

파일명

SafeArchive.md

섹션 순서

1. 추천 보관 정책
2. 추천 보관함
3. 보관 기간
4. 삭제 가능 여부
5. 사용자 확인 필요 여부
6. 보관 이유

분량

250~500자

---

## 성공 기준

* [ ] 즉시 삭제를 제안하지 않는다.
* [ ] 메일 유형에 맞는 보관함을 추천한다.
* [ ] 보관 기간을 제공한다.
* [ ] 사용자 확인 필요 여부를 표시한다.
* [ ] 오탐 방지 원칙을 반영한다.
* [ ] Action Recommendation Skill에서 사용할 수 있는 결과를 생성한다.
