## 목적 및 범위

Mail Classification, Priority Analysis, Security Check, Newsletter Manager 결과를 기반으로
이메일을 즉시 삭제하지 않고 안전하게 보관하는 정책을 제안한다.

결과물

* ## 추천 보관 정책
* ## 추천 보관함
* ## 보관 기간
* ## 삭제 가능 여부
* ## 사용자 확인 필요 여부
* ## 보관 이유

6개 섹션으로 구성된 `.md` 파일

---

## 작업 절차

1. spec.md에서 목표, 출력 형식, 성공 기준을 읽는다.
2. Mail Classification Skill 결과를 읽는다.
3. Priority Analysis Skill 결과를 읽는다.
4. Security Check Skill 결과를 읽는다.
5. Newsletter Manager Skill 결과를 읽는다.
6. 메일의 유형, 중요도, 보안 위험도를 종합한다.
7. 메일 유형에 따라 추천 보관함을 결정한다.
8. 보관 기간을 결정한다.
9. 삭제 가능 여부와 사용자 확인 필요 여부를 작성한다.
10. 보관 이유를 작성한다.
11. 성공 기준과 결과물을 비교하여 누락된 항목이 없는지 확인한다.
12. 다음 Skill이 사용할 수 있도록 "다음 Skill 전달 정보" 섹션을 작성한다.
13. 정보가 불확실한 경우 "확인 필요"로 표시한다.
14. 성공 기준과 결과물을 비교하여 누락된 항목이 없는지 확인한다.
---

## 사용 규칙 및 제약 사항

* 어떤 경우에도 즉시 삭제를 제안하지 않는다.
* 삭제는 사용자 최종 확인 후 가능하다고 안내한다.
* 중요 메일은 받은편지함 유지를 추천한다.
* 피싱 의심 메일은 위험 메일함에 30일 격리 보관을 추천한다.
* 광고 메일은 광고함에 14일 보관을 추천한다.
* 뉴스레터는 뉴스레터함에 30일 보관을 추천한다.
* 불확실한 메일은 검토 필요함에 7일 보관을 추천한다.
* Markdown 형식을 유지한다.
