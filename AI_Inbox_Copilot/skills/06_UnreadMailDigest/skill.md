## 목적 및 범위

Mail Summary, Mail Classification, Priority Analysis, Security Check의 결과를 기반으로 읽지 않은 메일을 분석하여 중요한 메일만 요약한다.

결과물

* ## 읽지 않은 메일 현황
* ## 반드시 읽어야 하는 메일
* ## 답장이 필요한 메일
* ## 마감이 있는 메일
* ## 피싱 의심 메일
* ## 나중에 확인해도 되는 메일

6개 섹션으로 구성된 `.md` 파일

---

## 작업 절차

1. spec.md에서 목표, 출력 형식, 성공 기준을 읽는다.
2. Mail Summary Skill 결과를 읽는다.
3. Mail Classification Skill 결과를 읽는다.
4. Priority Analysis Skill 결과를 읽는다.
5. Security Check Skill 결과를 읽는다.
6. 읽지 않은 메일 목록을 확인한다.
7. 중요도가 높은 메일을 우선 선별한다.
8. 답장이 필요한 메일을 분류한다.
9. 마감일이 있는 메일을 추출한다.
10. 피싱 의심 메일을 별도로 분류한다.
11. 광고 및 뉴스레터는 하나의 그룹으로 묶는다.
12. Markdown 형식으로 결과를 생성한다.
13. 성공 기준과 결과물을 비교하여 누락된 항목이 없는지 확인한다.
14. 다음 Skill이 사용할 수 있도록 "다음 Skill 전달 정보" 섹션을 작성한다.
15. 정보가 불확실한 경우 "확인 필요"로 표시한다.
16. 성공 기준과 결과물을 비교하여 누락된 항목이 없는지 확인한다.
---

## 사용 규칙 및 제약 사항

* 모든 읽지 않은 메일을 출력하지 않는다.
* 중요도가 높은 메일을 우선 제공한다.
* Priority Analysis 결과를 반드시 따른다.
* Security Check 결과를 반드시 반영한다.
* 광고와 뉴스레터는 개별 출력하지 않고 그룹화한다.
* 메일을 삭제하거나 이동하지 않는다.
* Morning Mail Brief에서 사용할 수 있도록 명확한 결과를 제공한다.
* Markdown 형식을 유지한다.
