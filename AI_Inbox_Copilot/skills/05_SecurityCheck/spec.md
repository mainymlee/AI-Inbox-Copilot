# spec.md — Security Check

## 목표

> Mail Summary, Mail Classification, Priority Analysis 결과를 기반으로 이메일의 보안 위험도를 분석하여 피싱, 스팸, 사칭, 악성 링크 및 위험 첨부파일 여부를 판단한다.

---

## 맥락

### 대상

* 대학생
* 직장인
* 이메일을 자주 사용하는 일반 사용자

### 사용 목적

사용자가 피싱 메일이나 악성 메일을 쉽게 식별하고
안전한 대응을 할 수 있도록 돕는다.

---

## 범위

### 입력

* Mail Summary 결과
* Mail Classification 결과
* Priority Analysis 결과
* 메일 제목
* 메일 본문
* 발신자
* 링크 정보
* 첨부파일 정보

### 출력

Markdown 형식의 보안 분석 결과

포함 내용

* 메일 정보
* 보안 위험도(0~100%)
* 위험 요소
* 피싱 가능성
* 추천 행동

---

## 제약

* 보안 위험도는 반드시 0~100%로 표시한다.
* 피싱 여부를 단정하지 않고 "가능성"으로 표현한다.
* 위험 요소를 반드시 설명한다.
* 위험 링크 및 첨부파일이 의심되면 경고 문구를 포함한다.
* 메일을 삭제하거나 격리하지 않는다.
* Markdown 형식을 유지한다.
- 모든 출력은 공통 출력 형식의 "다음 Skill 전달 정보" 섹션을 포함한다.
- 다음 Skill이 사용할 수 있도록 핵심값을 명확한 항목명으로 작성한다.
- 없는 정보는 추측하지 않고 "없음" 또는 "확인 필요"로 표시한다.

## 보안 위험도 기준

Security Score는 0~100%로 계산한다.

### 위험도 가산 기준

| 조건 | 점수 |
|---|---:|
| 의심스러운 발신자 주소 | +20 |
| 공식 기관 또는 기업 사칭 가능성 | +20 |
| 긴급성 유도 표현 포함 | +15 |
| 개인정보 입력 요구 | +20 |
| 비밀번호 변경 요구 | +15 |
| 결제 또는 송금 요구 | +20 |
| 단축 URL 포함 | +15 |
| 도메인이 공식 도메인과 유사함 | +20 |
| 첨부파일 실행 유도 | +15 |
| 문법적으로 어색한 표현 | +10 |
| 과도한 혜택 또는 당첨 안내 | +10 |

### 위험도 감점 기준

| 조건 | 점수 |
|---|---:|
| 신뢰 가능한 발신자 | -20 |
| 사용자가 이전에 자주 주고받은 발신자 | -15 |
| 공식 도메인 확인 가능 | -20 |
| 개인정보나 결제 요구 없음 | -10 |

### 점수 보정

- 최종 점수는 0% 미만으로 내려가지 않는다.
- 최종 점수는 100%를 초과하지 않는다.
- 여러 조건에 해당하면 점수를 합산한 뒤 0~100 범위로 보정한다.

### Risk Level 기준

| Security Score | Risk Level | 의미 |
|---:|---|---|
| 0~19 | SAFE | 일반적으로 안전 |
| 20~39 | LOW | 낮은 위험 |
| 40~59 | MEDIUM | 주의 필요 |
| 60~79 | HIGH | 위험 가능성 높음 |
| 80~100 | CRITICAL | 매우 위험, 즉시 주의 필요 |

## 출력 형식

파일명

SecurityCheck.md

1. 메일 정보
