# spec.md — Morning Mail Brief

## 목표

> 여러 Mail Skill의 분석 결과를 종합하여 사용자가 하루를 시작하기 전에 반드시 확인해야 하는 메일과 해야 할 일을 Morning Mail Brief 형태로 제공한다.

---

## 맥락

### 대상

- 대학생
- 직장인
- 이메일을 자주 사용하는 일반 사용자

### 사용 목적

메일을 하나씩 확인하지 않아도
오늘 가장 중요한 메일과 해야 할 일을
30초 안에 파악할 수 있도록 돕는다.

---

## 범위

### 입력

- Mail Classification 결과
- Priority Analysis 결과
- Security Check 결과
- Unread Mail Digest 결과
- Newsletter Manager 결과
- Safe Archive 결과
- Action Recommendation 결과

### 출력

Markdown 형식의 Morning Mail Brief

포함 내용

- 오늘의 메일 현황
- 중요 메일 TOP3
- 오늘의 일정
- 답장이 필요한 메일
- 피싱 의심 메일
- 뉴스레터/광고 현황
- Inbox Health Score
- 오늘 해야 할 일

---

## 제약

- Markdown 형식으로 출력한다.
- 메일 내용을 그대로 복사하지 않는다.
- 사용자가 30초 안에 이해할 수 있도록 작성한다.
- 중요 메일은 이유를 함께 설명한다.
- 피싱 메일은 반드시 경고를 포함한다.
- Safe Archive 정책을 위반하지 않는다.
- Action Recommendation 결과를 반드시 반영한다.
- 모든 출력은 공통 출력 형식의 "다음 Skill 전달 정보" 섹션을 포함한다.
- 다음 Skill이 사용할 수 있도록 핵심값을 명확한 항목명으로 작성한다.
- 없는 정보는 추측하지 않고 "없음" 또는 "확인 필요"로 표시한다.
---

## 출력 형식

파일명

MorningMailBrief.md

섹션 순서

1. 오늘의 메일 현황
2. 중요 메일 TOP3
3. 오늘 일정
4. 답장이 필요한 메일
5. 피싱 의심 메일
6. 뉴스레터 및 광고
7. Inbox Health Score
8. 오늘 해야 할 일

분량

500~800자

---

## Inbox Health Score 기준

Inbox Health Score는 0~100점으로 계산한다.

기본 점수는 100점에서 시작한다.

### 감점 기준

| 조건 | 감점 |
|---|---:|
| 읽지 않은 중요 메일 1개당 | -10 |
| 답장 필요한 미처리 메일 1개당 | -10 |
| 오늘 마감되는 미처리 메일 1개당 | -15 |
| 피싱 의심 메일 1개당 | -20 |
| 장기간 읽지 않은 뉴스레터 5개당 | -5 |
| 검토 필요 메일 1개당 | -5 |

### 점수 보정

- 최종 점수는 0점 미만으로 내려가지 않는다.
- 최종 점수는 100점을 초과하지 않는다.

### 점수 등급

| 점수 | 상태 |
|---:|---|
| 90~100 | 매우 좋음 |
| 70~89 | 양호 |
| 50~69 | 주의 |
| 30~49 | 정리 필요 |
| 0~29 | 즉시 정리 필요 |

## 성공 기준

- [ ] 다른 Skill의 결과를 모두 반영한다.
- [ ] 중요 메일을 우선순위대로 정렬한다.
- [ ] 오늘의 일정을 포함한다.
- [ ] 답장이 필요한 메일을 제공한다.
- [ ] 피싱 의심 메일을 별도로 표시한다.
- [ ] Inbox Health Score를 계산한다.
- [ ] Action Recommendation을 반영한다.
- [ ] 사용자가 30초 안에 이해할 수 있다.
