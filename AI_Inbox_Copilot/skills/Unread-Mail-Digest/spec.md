# spec.md — Unread Mail Digest

## 목표

> 일정 기간 동안 읽지 않은 이메일을 분석하여 사용자가 우선적으로 확인해야 하는 메일을 선별하고 요약한다.

---

## 맥락

### 대상

* 대학생
* 직장인
* 읽지 않은 메일이 많이 쌓이는 사용자

### 사용 목적

읽지 않은 메일을 모두 확인하지 않아도
중요한 메일과 놓치면 안 되는 메일을 빠르게 파악하도록 돕는다.

---

## 범위

### 입력

* Mail Summary 결과
* Mail Classification 결과
* Priority Analysis 결과
* Security Check 결과
* 읽지 않은 메일 목록

### 출력

Markdown 형식의 읽지 않은 메일 요약

포함 내용

* 읽지 않은 메일 현황
* 반드시 읽어야 하는 메일
* 답장이 필요한 메일
* 마감이 있는 메일
* 피싱 의심 메일
* 나중에 확인해도 되는 메일

---

## 제약

* 읽지 않은 메일 전체를 나열하지 않는다.
* Priority Analysis 결과를 반드시 반영한다.
* Security Check 결과를 반드시 반영한다.
* 광고 메일은 하나의 그룹으로 묶는다.
* 메일 삭제나 이동을 수행하지 않는다.
* Markdown 형식을 유지한다.
- 모든 출력은 공통 출력 형식의 "다음 Skill 전달 정보" 섹션을 포함한다.
- 다음 Skill이 사용할 수 있도록 핵심값을 명확한 항목명으로 작성한다.
- 없는 정보는 추측하지 않고 "없음" 또는 "확인 필요"로 표시한다.
---

## 출력 형식

파일명

UnreadMailDigest.md

섹션 순서

1. 읽지 않은 메일 현황
2. 반드시 읽어야 하는 메일
3. 답장이 필요한 메일
4. 마감이 있는 메일
5. 피싱 의심 메일
6. 나중에 확인해도 되는 메일

분량

400~700자

---

## 성공 기준

* [ ] 읽지 않은 메일 현황을 제공한다.
* [ ] 반드시 읽어야 하는 메일을 선별한다.
* [ ] 답장이 필요한 메일을 제공한다.
* [ ] 마감이 있는 메일을 제공한다.
* [ ] 피싱 의심 메일을 별도로 표시한다.
* [ ] 광고 메일은 그룹화하여 제공한다.
* [ ] Morning Mail Brief에서 사용할 수 있는 결과를 생성한다.
