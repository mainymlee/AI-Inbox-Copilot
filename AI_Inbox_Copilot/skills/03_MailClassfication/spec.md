# spec.md — Mail Classification

## 목표

> 이메일의 내용을 분석하여 적절한 카테고리로 분류하고 이후 Skill들이 활용할 수 있는 분류 결과를 생성한다.

---

## 맥락

### 대상

* 대학생
* 직장인
* 이메일을 자주 사용하는 일반 사용자

### 사용 목적

메일을 자동으로 분류하여
중요도 분석, 보안 분석, 뉴스레터 관리 등
후속 Skill이 활용할 수 있도록 한다.

---

## 범위

### 입력

* Mail Summary 결과
* 메일 제목
* 메일 본문
* 발신자

### 출력

Markdown 형식의 메일 분류 결과

포함 내용

* 메일 카테고리
* 분류 이유
* 신뢰도(Confidence)

---

## 제약

* 하나 이상의 카테고리를 선택할 수 있다.
* 반드시 분류 이유를 작성한다.
* 확신이 낮은 경우 "기타"를 포함한다.
* Markdown 형식을 유지한다.
- 모든 출력은 공통 출력 형식의 "다음 Skill 전달 정보" 섹션을 포함한다.
- 다음 Skill이 사용할 수 있도록 핵심값을 명확한 항목명으로 작성한다.
- 없는 정보는 추측하지 않고 "없음" 또는 "확인 필요"로 표시한다.
---

## 출력 형식

파일명

MailClassification.md

섹션 순서

1. 메일 정보
2. 분류 결과
3. 분류 이유
4. 신뢰도

분량

200~300자

---

## 성공 기준

* [ ] 메일을 하나 이상의 카테고리로 분류한다.
* [ ] 분류 이유를 제공한다.
* [ ] 신뢰도를 제공한다.
* [ ] Markdown 형식을 유지한다.
* [ ] 후속 Skill이 사용할 수 있는 결과를 생성한다.
