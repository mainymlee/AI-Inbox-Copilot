## 목적 및 범위

Mail Summary와 Mail Classification 결과를 기반으로
뉴스레터와 광고 메일을 분석하여 관리 방안을 제안한다.

결과물

* ## 뉴스레터 현황
* ## 발신자별 그룹
* ## 장기간 읽지 않은 뉴스레터
* ## 추천 관리 방법
* ## 주간 요약 제안

5개 섹션으로 구성된 `.md` 파일

---

## 작업 절차

1. spec.md에서 목표, 출력 형식, 성공 기준을 읽는다.
2. Mail Summary Skill 결과를 읽는다.
3. Mail Classification Skill 결과를 읽는다.
4. 뉴스레터 및 광고 메일을 식별한다.
5. 발신자별로 메일을 그룹화한다.
6. 최근 열람 기록을 분석하여 장기간 읽지 않은 뉴스레터를 찾는다.
7. 발신 빈도를 분석한다.
8. 다음 관리 방법 중 적절한 방법을 추천한다.

   * 계속 구독
   * 주간 요약으로 확인
   * 관리 대상 추가
   * 구독 해지 검토
9. Markdown 형식으로 결과를 생성한다.
10. 성공 기준과 결과물을 비교하여 누락된 항목이 없는지 확인한다.
11. 다음 Skill이 사용할 수 있도록 "다음 Skill 전달 정보" 섹션을 작성한다.
12. 정보가 불확실한 경우 "확인 필요"로 표시한다.
13. 성공 기준과 결과물을 비교하여 누락된 항목이 없는지 확인한다.
---

## 사용 규칙 및 제약 사항

* 메일을 삭제하거나 이동하지 않는다.
* 구독 해지를 직접 수행하지 않는다.
* 구독 해지는 추천만 한다.
* 발신자별로 그룹화한다.
* 최근 열람 기록을 함께 고려한다.
* Safe Archive Skill에서 활용할 수 있도록 결과를 제공한다.
* Markdown 형식을 유지한다.

## 템플릿

```markdown
# 📰 Newsletter Manager

## 뉴스레터 현황

- 전체 뉴스레터:
- 광고 메일:
- 주요 발신자:

---

## 발신자별 그룹

| 발신자 | 메일 수 | 최근 열람 | 추천 |
|--------|---------|-----------|------|
|        |         |           |      |

---

## 장기간 읽지 않은 뉴스레터

| 발신자 | 마지막 열람 | 추천 |
|--------|-------------|------|
|        |             |      |

---

## 추천 관리 방법

- 계속 구독
- 주간 요약으로 확인
- 관리 대상 추가
- 구독 해지 검토

---

## 주간 요약 제안

이번 주 뉴스레터를 하나의 요약으로 제공하도록 추천합니다.
```
